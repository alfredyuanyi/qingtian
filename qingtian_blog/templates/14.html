在.NET Framework中，一般规则是要变量名要反映变量实例的功能而不是反映数据类型。 名称的约定包括以下几个方面：

  1. 名称的大小写
    名称空间，类，方法，属性采用Pascal命名，即名称中单词的首字母偶读大写。如：
      		class  EmployeeSalary
		public void OpenFile()
		public string Name
    类中的私有成员（一般加前缀_)，传递给方法的参数 一般采用camel命名法，即名称的第一个单词的首字母不大写，其余的单词的首字母大写，如
      		private int _studentId
		public void OpenFile(string filePath)
  2. 名称的风格
    名称的风格应该一致。例如，类中的一个方法被命名为ShowFileDialog()，另一个方法就不能被命名为ShowDialogWarning()或者WarningDialogShow()，而应该是ShowWarningDialog()
  3. 名称空间的名称
    名称空间的名称是.NET区分共享程序集中对象名的唯一方式。最好是用自己的公司名创建顶级的名称空间，再嵌套低一级的名称空间推荐如下：
    <CompanyName>.<TechnologyName>
  4. 名称和关键字
    名称不能和任何关键字冲突。
  5. 属性和方法的使用
    • 客户端代码应能读取它的值，最好不要使用只写属性，例如，应使用SetPassword()方法，而不是Password只写属性
    • 读取改制不应花太长的时间。实际上，如果它是一个属性，通常表明读取过程画的时间相对较短。
    • 读取该值不应有任何明显的和不希望的负面效应。设置属性的值，不应由于该属性不直接相关的负面效应。谁知对话框的宽度会改变该对话框在屏幕上的外观，这是可以的，因为它与该属性相关。
    • 可以按照任何顺序设置属性。
    • 顺序读取属性也应有相同的效果
    如果编码的相关想满足上述的所有条件，那就把它设置为属性，否则就应该使用方法。
  6. 字段的用法
    字段应总是私有的，某些情况下也可以把常量或只读字段设置为共有的，原因是如果把字段设置为公有就不利于以后扩展或修改类
