把一个大的整数类型转换成浮点类型会保留整数部分，但是有时会丢失精度这是因为浮点类型总是有比整数类型更大的值，但是可能只有更少的精度。
举个例子：

	#region  用float会丢失精度，用double则不会
	int i = 100000001;
	float f = i;
	int j = (int)f;
	Console.WriteLine("i = {0};  f = {1}; j = {2}",i,f,j);
	#endregion
运行结果：
i = 100000001; f = 1E+08; j = 100000000

解释：
因为float的精度只到七，所以f在表示的时候是1.0000000E8 ，这样就把1给丢 了。。
