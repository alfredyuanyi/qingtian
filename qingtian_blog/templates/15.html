C#预处理器指令在调试代码的时候作用较大，但是实际销售软件的时候一般不希望编译这部分代码。
C#预处理器指令主要有：#define，#undef，#if，#endif，#elif（=else if），#else，#warning，#error，#region，#endregion（预处理器指令的开头都有符号#）

考虑下面的代码：

#define DEBUG  //  这类似于声明了一个变量
using System;
namespace hello
{
    
    class Program
    {
        public static void Main(string[] args)
        {
            int x = 0;
            x+=1;
             #if DEBUG
             Console.WriteLine(x);
             #endif
            Console.ReadKey();
        }
}
运行结果为：
1

实际上当程序有#define DEBUG时，就会编译#if的内容，当添加#undef指令后，程序将不再编译#if中的内容。这项技术称为条件编译。
